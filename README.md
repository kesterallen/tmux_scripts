# tmux_scripts
Script and cronjobs to record the state of a tmux session(s) and recreate it.

## Usage
Add the crontab line from crontab.txt to your crontab, and then tmux_state.py
can generate a script to recreate a recorded tmux session.

        Take the output from
            tmux list-p -aF "#{session_created};#S;#I;#P;#W;#{pane_current_path};#{pane_pid}"
        which is run by the list_tmux_panes method in RECORD mode

        e.g.:
            123;scorpius;1;1;gcloud;/home/kester/Dropbox/gcloud/projects/sf_p3;4190
            123;scorpius;1;2;gcloud;/home/kester/Dropbox/export/apple_health_export;4197
            123;scorpius;2;3;analytics;/home/kester;4221
            123;scorpius;2;4;analytics;/home/kester;169213
            123;scorpius;3;1;meetings;/home/kester;4231
            123;scorpius;4;1;codility;/home/kester/codility;4237
            123;scorpius;5;1;discord_music;/home/kester/Desktop;4248
            123;scorpius;5;2;discord_music;/home/kester/src/ayb-bot;4265
            123;scorpius;5;3;discord_music;/home/kester/src;23582
            123;scorpius;6;1;footpedal;/home/kester;4276
            123;scorpius;7;1;hermes robot protocol docs;/home/kester/src/hermes;4294
            123;scorpius;8;1;bash;/home/kester;299586

        and modify that to include the command that is generating the process ID
        (pane_pid) argument as a new final column (and CSV-ify it):
            123,scorpius,1,1,gcloud,/home/kester/Dropbox/gcloud/projects/sf_p3,4190,vim surlyfritter/scripts/wordle.py
